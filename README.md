# Serverless Task Manager API

[![CI](https://github.com/kumaraharsh84/serverless_task_manager_api/actions/workflows/ci.yml/badge.svg)](https://github.com/kumaraharsh84/serverless_task_manager_api/actions/workflows/ci.yml)
[![Python 3.12](https://img.shields.io/badge/python-3.12-blue.svg)](https://www.python.org/downloads/)
[![AWS SAM](https://img.shields.io/badge/AWS%20SAM-serverless-orange.svg)](https://aws.amazon.com/serverless/sam/)

A production-style serverless REST API for managing tasks on AWS with Lambda, API Gateway, DynamoDB, and SAM.

## Architecture

```text
Client
  |
  v
API Gateway
  |
  v
Lambda Functions
  |
  v
DynamoDB
```

## Features

- Full CRUD task operations
- Optional token-based authentication with `x-api-key`
- Filtering by task status
- Search across title and description
- Cursor-based pagination with `limit` and `lastKey`
- Soft delete with audit fields
- Structured logging for CloudWatch
- Infrastructure as Code with AWS SAM
- Automated test and validation workflow with GitHub Actions

## Task Model

| Field | Type | Notes |
| --- | --- | --- |
| `taskId` | string | UUID generated by the API |
| `title` | string | Required |
| `description` | string | Optional |
| `status` | string | `todo`, `in-progress`, `done` |
| `priority` | string | `low`, `medium`, `high` |
| `dueDate` | string | ISO-8601 datetime |
| `createdAt` | string | ISO-8601 datetime |
| `updatedAt` | string | ISO-8601 datetime |
| `deletedAt` | string | Present after soft delete |
| `isDeleted` | boolean | Soft delete flag |

## API Endpoints

| Method | Endpoint | Purpose |
| --- | --- | --- |
| `POST` | `/tasks` | Create a task |
| `GET` | `/tasks` | List tasks |
| `PUT` | `/tasks/{taskId}` | Update a task |
| `DELETE` | `/tasks/{taskId}` | Soft-delete a task |

### Query parameters for `GET /tasks`

| Parameter | Description | Default |
| --- | --- | --- |
| `status` | Filter by `todo`, `in-progress`, or `done` | none |
| `q` | Search text in title and description | none |
| `limit` | Number of items to return | `10` |
| `lastKey` | Pagination cursor from previous response | none |

## Build and Deploy

```powershell
sam build
sam deploy --guided
```

Recommended guided deploy values:

- Stack name: `internship-task-api`
- Region: `ap-south-1`
- Allow IAM role creation: `Y`
- Save configuration: `Y`
- `ApiToken`: set a secret value to enable auth, or leave blank to disable it

## Test the deployed API

```powershell
$api = "https://YOUR_API_ID.execute-api.ap-south-1.amazonaws.com/v1"
$headers = @{
  "x-api-key"    = "YOUR_TOKEN"
  "Content-Type" = "application/json"
}
```

Create a task:

```powershell
Invoke-RestMethod -Method POST -Uri "$api/tasks" -Headers $headers -Body '{"title":"Build project","description":"Internship work","status":"todo","priority":"high","dueDate":"2026-03-20T12:00:00+05:30"}'
```

List tasks:

```powershell
Invoke-RestMethod -Method GET -Uri "$api/tasks?status=todo&q=project&limit=5" -Headers $headers
```

Update a task:

```powershell
Invoke-RestMethod -Method PUT -Uri "$api/tasks/<TASK_ID>" -Headers $headers -Body '{"status":"in-progress","priority":"medium"}'
```

Delete a task:

```powershell
Invoke-RestMethod -Method DELETE -Uri "$api/tasks/<TASK_ID>" -Headers $headers
```

If you deploy without `ApiToken`, remove `-Headers $headers` from the commands above.
